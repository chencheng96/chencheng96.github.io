JavaScript,cookie

document对象中的cookie属性是存储在访问者计算机中的变量，保存在客户端的机器上，以便于服务器端程序在一个页面中使用另一个页面的数据，或者在用户离开页面并返回时恢复用户的状态变量。

同一台计算机通过浏览器请求某个页面时，会发生这个cookie。可以使用JavaScript来创建和取回cookie值。

cookie

cookie机制将信息存储于用户硬盘，可以作为全局变量。

创建和读取cookie

对cookie读操作可以得到一个字符串，这个字符串包含了应用到当前文档的所有的cookie的名字和值。通过设置cookie的值可以创建，修改或删除一个cookie。

除名字外，每个cookie有4个可选属性，分别控制生存期，可见性和安全性。

cookie的属性设置成一个字符串形式：name=value

document.cookie="name=value";        //将上述语句保存在cookie中

document.cookie="userid=100; username=chen";  //同时存储多个名值，用分号加空格隔开 尽量分开写！！！！

name为cookie的名字，value为cookie的值。

document.cookie="str="+escape("I Love You");  相当于document.cookie="str=I%20Love%You";

unescape(document.cookie);      //进行解码得到原来的cookie值。

cookie的名或值中不能有分号，逗号，等号以及空格。如果要使用这些特殊符号，必须在写入cookie前，使用escape()函数对cookie值进行编码，它能将一些特殊符号用16进制表示，读取cookie时再通过unescape()函数将其还原。

在同一网页创建的cookie，都会写在一个cookie文件中。读cookie时，只会读cookie文件的所有内容。

document.cookie    //读取cookie内容

改写cookie的值，只要使用相同名字的cookie名和新的值重新设置一次就行了。

<script type="text/javascript">
function setcookie(){
document.cookie="userid="+document.form1.userid.value;
document.cookie="username="+document.form1.username.value;
window.alert("cookie中的保存的值是：\n"+document.cookie);
}
</script>
<form name="form1">
用户id：<input type="text" name="userid"/></br>
用户名字：<input type="text" name="username"/></br>
<input type="button" onclick="setcookie()" value="保存cookie"/></br>
</form>

获取cookie的值

只能获取所有的cookie值，而不能指定cookie名字来获得指定的值。

<script type="text/javascript">
document.cookie="userid=000001";
document.cookie="username=chen";
document.cookie="userpass=111111";
var str=document.cookie;      //读取cookie
document.write("cookie中的保存的值是：</br>"+str);
var arrcookie=str.split("; ");     //分割cookie，保存在数组中。这里分割要用“; ”分号加空格，不然会出错。
for(var i=0;i<arrcookie.length;i++){
document.write("<br/>*****"+i+arrcookie[i]);
}
for(var i=0;i<arrcookie.length;i++){
var arr=arrcookie[i].split("=");      //分割cookie名和cookie值
if("userid"==arr[0]){
var userid=arr[1];
break;
}
}
window.alert("userid的值是："+userid);
</script>

cookie的生存期（expires）

cookie的第一个性质是生存期（expires）。默认情况下，cookie是暂时存在的，他们存储的值只是在浏览器会话期间存在，当用户退出浏览器后就会丢失。可以用expires性质指定一个终止日期，这样就会使浏览器把cookie保存到一个本地文件中，以便用户下次访问这个网页能读出。一但超过规定时间，cookie会自动从cookie文件中删除。

<script type="text/javascript">
var d=new Date();
var expiredays=1;      //一天后过期
d.setTime(d.getTime()+expiredays*24*3600*1000);
document.cookie="username=chen"+d.toGMTString();
window.alert("cookie中的保存的值是：\n"+document.cookie);
</script>

cookie的路径（path）

指定了与cookie关联在一起的网页。默认情况下，cookie会和创建它的网页以及与这个网页处于同一目录下的网页和处于该目录的子目录的子目录下的网页关联在一起。

http://www.123.com/book.3.html

那么该网站的/book/虚拟目录以及该虚拟目录下的所有子目录都可以读取该cookie。

cookie的域名（domain）

解决在不同域中访问cookie的问题。

path=/;domin=.123.com

通常在设置cookie的domain时，都会将cookie的path设为“/”。如果当前文件在www.123.com域上，只能设置“.123.com”而不能是其他域

cookie的安全性（secure）

cookie的secure是一个布尔类型。设置之后cookie只能通过HTTPS或在其他安全的协议下才能传输

cookie的局限性

  • 浏览器保存的cookie总数不能超过300个
  • 为每一个web服务器保存的cookie数不超过20个
  • 每个cookie存放的数据不超过4KB
  • cookie的生存周期是以毫秒为单位计算的
  • 用户浏览器可通过设置来接受或拒绝cookie
