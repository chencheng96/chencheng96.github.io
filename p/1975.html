本次使用环境Windows Server 2019,需要使用Exchange 2019管理端。

在部署Exchange之前，需要在准备部署Exchange 2019 管理端的服务器上添加必须的角色和功能，同时加入到AD，并以域管理员账号登录。

打开服务器管理器，在服务器管理器页面点击添加功能和角色，打开添加角色和功能向导，在该页面默认下一步即可。

在安装类型页面选择“基于角色或基于功能的安装”

在服务器选择页面，选择要安装的服务器

在服务器角色页面，选择Web服务器（IIS）

在选择功能页面，点击要添加的功能

在Web服务器角色（IIS）页面，保持默认下一步即可

在角色服务页面，选择管理勾选管理工具，（IIS管理控制台、IIS管理兼容性、IIS元数据库兼容性、IIS WMI兼容性、IIS管理控制台）,并点击下一步

在确认页面点击安装即可。

完成角色添加完毕之后，直接关闭页面即可。

安装必须的软件包

  • 安装Visual C++ Redistributable Package for Visual Studio 2012
  • 安装Visual C++ Redistributable Package for Visual Studio 2013

准备完毕后，即可进行部署Exchange管理端。将ExchangeISO挂载到安装的服务器，点击介质的setup.exe,即可进入安装向导，在是否检查更新页面，点击现在不检查更新，然后下一步即可

单击下一步按钮后，显示如图所示的“正在复制文件”对话框，复制必须的文件到管理计算机中。

文件复制完毕后，显示如图所示的正在初始化安装程序窗口

初始化安装程序后，打开Exchange简介窗口

点击完“下一步”按钮，阅读许可协议，并勾选我接受许可协议中的条款

点击下一步，进入推荐设置页面，在此我们选择不使用推荐设置

点击“下一步”按钮后，进入到服务器角色选择页面，在此我们选择“管理工具”即可

点击下一步后，进入到“安装空间和位置”页面，在此我们自定义选择路径

系统将进入准备情况检查阶段，检查无误后，点击“安装”即可

进入安装进度页面

经过漫长的等待，管理工具完成安装，点击完成即可。

重启计算机，在windows开始按钮，会多出如下几个文件夹

Exchange管理工具完成安装。
