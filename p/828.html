Eclipse如何导出安卓apk文件

thefirst在Eclipse中选一个Android工程，然后鼠标右键选 Export
选择 Android ->Export Android Application 
之前已经选择了工程，点击 Next 
若没keystore，选择 Create new keystore ，然后指定路径和密码 
输入 Alias 名称和密码，指定其有效期。界面下面至少输一个
输入要导出 apk 的名称，点Finish 。这样就得到了一个已经签名的apk。

Android问：Dx unsupported class file version 52.0

Dx unsupported class file version 52.0
...while parsing com/nn/windows/WMain.class
上述报错后跟着报错：Conversion to Dalvik format failed with error 1


Conversion to Dalvik format failed with error 1是Dx unsupported class file version 52.0的错误引起的。是由于Java编译器版本和Android sdk版本不匹配。

解决方法：把java的编译器改为1.6版本的（Android sdk版本也是1.6的话）。右键项目->properties->Java compiler

怎么解决新建项目的报错No resource found that matches the give


AndroidManifest.xml:11: error: Error: No resource found that matches the given name (at 'icon' with value '@drawable/ic_launcher') 错误发生在AndroidManifest.xml中的android:icon="@drawable/ic_launcher"


Eclipse Neon版本新建的android项目使用的图标都是使用mipmap文件夹里面的，就是那个ic_launcher系统默认机器人在这个文件夹里面，app内置图片都放在drawable中，所以可以直接修改图片来源是mipmap 而不是drawable。

直接把drawable改成mipmap就行，你的@的文件不在drawable就在mipmap里

Eclipse中ADT，sdk，dk，各是干什么的


1、JDK(Java Development Kit)是 Java 语言的软件开发工具包(SDK)。开发Java的工具包。

2、软件开发工具包（外语首字母缩写：SDK、外语全称：Software Development Kit）是为特定软件包、软件框架、硬件平台、操作系统等建立应用软件时开发工具的集合。

3、安卓开发工具---Android Develop Tool。ADT是Android开发的eclipse插件，用于打包和封装Android应用

Hardcoded string 汉字, should use @string resource警告

在一个RelativeLayout布局中加入一个Button按钮，在android:text="确定"处总出现一个提示“Hardcoded string "确定", should use @string resource”



应该在res/values/strings.xml中设置：

<!--?xml version="1.0" encoding="utf-8"?-->

<resources><string name="message">下一步</string></resources>

引用的时候使用 android:text="@string/message"就行了。
