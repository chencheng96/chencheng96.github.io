由于Oracle的庞大，有时候我们需要在只安装Oracle客户端如plsql、toad等的情况下去连接远程数据库，可是没有安装Oracle就没有一切的配置文件去支持。最后终于发现一个很有效的方法，Oracle的Instant client工具包可以很好地解决这个问题，而且小而方便。
1、首先到Oracle网站下载Instant（官网需要注册）。注意下载32位的instantclient，否则PLSQL无法正常连接。
2、在D:/instantclient_11_2目录下新建目录network,在network目录下再建admin目录,在admin目录下新建文件tnsnames.ora（评论中有介绍），这样选择数据库可以直接在下拉列表中选择了。
3、下载并安装PL.SQL.Developer配置应用，配置tools->preferences->connection设置：

Oracle Home为：D:/instantclient_11_2
OCI library为：D:/instantclient_11_2/oci.dll

配置完成后关闭PL/SQL ,再重启。主机名就会出现在PL/SQL Developer的列表里，输入用户名密码，就可以登录远程oracle 数据库。

    tnsnames.ora文件配置

        2018-09-11    

    tnsnames.ora是plsql用的东西，没有特别用处，用处是：不用输入完整的oracle地址，用简易的字符串代替。如本该是10.10.15.103：1521/orcl，通过设置tnsnames.ora中，就可以用“15.103-ORCL”代替了。在oracl（绿色版100M以内）解压目录下新建目录network,在network目录下再建admin目录,在admin目录下新建文件tnsnames.ora,打开写入如下内容: 

    # tnsnames.ora Network Configuration File: E:\oracle\product\10.2.0\client_1\network\admin\tnsnames.ora
# Generated by Oracle configuration tools.

ORCL =
  (DESCRIPTION =
    (ADDRESS_LIST =
      (ADDRESS = (PROTOCOL = TCP)(HOST = 10.141.11.111)(PORT = 1521))
    )
    (CONNECT_DATA =
      (SERVICE_NAME = ORCL)
    )
  )

JWDB =
  (DESCRIPTION =
    (ADDRESS_LIST =
      (ADDRESS = (PROTOCOL = TCP)(HOST = 202.114.11.111)(PORT = 1521))
    )
    (CONNECT_DATA =
      (SERVICE_NAME = JWDB)
    )
  )

    这样在打开PLSQL时，在Database的选项中就会出现上面定义的两个远程服务器。

    编码问题（待处理）

        2018-09-12    
    当我们连接成功后有时候查询出来的数据会出现乱码的问题，这是因为本地的编码和服务器端编码不一致，这时候我们可以通过SQL语句：select userenv('language') from dual;查询出服务器端的编码，如我自己的查询结果为

    USERENV('LANGUAGE')
    AMERICAN_AMERICA.ZHS16GBK

    我们就需要添加一个环境变量NLS_LANG ,值为: AMERICAN_AMERICA.ZHS16GBK 然后重启PL/SQL就不会再有乱码问题了。


    在PLSQL Developer目录下新建一bat文件 

    @echo off
set path=D:\oracle\product\instantclient_11_2
set ORACLE_HOME=D:\oracle\product\instantclient_11_2
set TNS_ADMIN=D:\oracle\product\instantclient_11_2
set NLS_LANG=AMERICAN_AMERICA.ZHS16GBK
start plsqldev.exe

    运行bat文件时，打开plsql再次查询数据时正常。
