该问题主要是由于当前操作的字符串是bytes类型的字符串对象，并对该bytes类型的字符串对象进行按照str类型的操作。
当对s进行按照str类型的操作（split）时，会弹出一下错误提示。因为split函数传入的参数是str类型的字符串，而实际的s为bytes编码类型的字符串对象

此时的解决办法是，
方法1：bytes-->str 将s转码成为str类型的或者是将对s进行str类型操作转化为对bytes类型的操作。

str和bytes类型之间的常用转码方式：

1、str to bytes:  #字符串a转换成字节b
b = bytes(a,encoding='utf-8')

b1 = bytes(a,encoding='gbk')
或
b = a.encode(encoding='utf-8')

b= a.encode(encoding='gbk')

2、bytes to str    #将字节b转换成字符c

c=str(b,encoding='utf-8')
c1=str(b1,encoding='gbk')
或
c = b.decode(encoding='utf-8')
c = b.decode()      #按gbk的方式编码

方法2：对s进行str类型操作转化为对bytes类型的操作
由于split里面用到分解字符串中的是空格‘  ’，但是该‘ ’是按照str编码的，需要转化成为bytes类型的，然后再直接对s进行对bytes类型的操作。


出现该错误的常见场景

1：查找文件时：
with open(fname, 'rb') as f:
if b'some-pattern' in tmp: continue
出现该错误往往是通过open()函数打开文本文件时，使用了‘rb’属性，如：fileHandle=open(filename,'rb'),则此时是通过二进制方式打开文件的，所以在后面处理时如果使用了str()函数，就会出现该错误，该错误不会再python2中出现。

具体解决方法有以下两种：
第一种，在open()函数中使用‘r’属性，即文本方式读取，而不是‘rb’,以二进制文件方式读取，可以直接解决问题。
第二种,在open()函数中使用‘rb’,可以在使用之前进行转换。
2：使用socket连接时：
import socket               # 导入 socket 模块
s = socket.socket()         # 创建 socket 对象
host = socket.gethostname() # 获取本地主机名
port = 12345                # 设置端口
s.bind((host, port))        # 绑定端口
s.listen(5)                 # 等待客户端连接
while True:
c, addr = s.accept()     # 建立客户端连接。
print ('连接地址：', addr)
stri='欢迎访问菜鸟教程！'   #需要转码否则会报错
b1=str.encode(stri)
c.send(b1)
c.close()                # 关闭连接
该场景一般将字节类型数据转换为字符串类型
3：提前进行编码：
with open(fname, 'rb') as f:
lines = [x.decode('utf8').strip() for x in f.readlines()]
