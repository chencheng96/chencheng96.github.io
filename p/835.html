转发和重定向的区别，怎么选择是重定向还是转发？

  • 2018-01-15
  • web html css js 插件
  • 公开
转发是服务器行为，重定向是客户端行为。转发时1次请求，重定向是2次。转发地址栏不会发生改变，重定向地址栏会改变。转发在项目内，重定向可以转到项目外。当使用转发时，将使用一个内部的方法来调用目标页面，新的页面继续处理同一个请求，而浏览器将不会知道这个过程。与之相反，重定向方式的含义是第一个页面通知浏览器发送一个新的页面请求。
通常情况下转发更快，而且能保持request内的对象。但是由于在转发之后，浏览器中URL仍然指向开始页面，此时如果重载当前页面，开始页面将会被重新调用。
