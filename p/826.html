项目直接放在webapps下，可以直接将webapps下文件替换掉。myeclipse中通过File-New-Web Project创建一个web项目，目录直接选择安装的apache-tomcat下的webapps目录，会自动创建一系列文件。

Java Web应用通过基于XML的发布描述来配置其发布信息，这个文件名字为web.xml，该文件应放在WEB-INF目录下，包含如下配置信息：

  • Servlet的定义
  • Servlet的初始化参数
  • Servlet以及JSP映射
  • Filter的定义
  • 安全域配置参数
  • welcome文件清单
  • 资源引用
  • 环境变量的定义

用标签操作JavaBean

  • <jsp:useBean>标签用来装载一个将在JSP页面中使用的JavaBean。<jsp:useBean id="name" class="package.class"/>
    • id：命名引用该bean变量。如果能找到id和scope相同的Bean实例，jsp:useBean动作将使用已有的Bean实例而不是创建新的。
    • class="classname"：指定Bean完整的包名。
    • scope="scopeType"：指定Bean在哪种上下文内可用，可用取值为：page,request,session,application
    • type="className"：指定引用该对象的变量类型
    • beanName="beanName"：指定Bean的名字，如果提供了type属性和beanName属性，允许省略class属性。
  • <jsp:getProperty>标签提取指定Bean属性的值，转换成字符串，然后输出。有两个必要的属性：
    • name：表示要设置属性的Bean
    • property：表示要设置的属性
  • <jsp:setProperty>用来设置已经实例化的Bean对象的属性，有4个属性
    • name：表示要设置的Bean
    • property：表示要设置的属性
    • value：指定Bean属性的值
    • param：指定用哪个请求参数作为Bean属性的值

value属性字符串数据会在目标类中通过标准的valueOf方法自动转换成数字，boolen，Boolean，byte，Byte，char，Character。boolen和Boolean类型属性通过Boolen.valueOf转换，int和Integer类型的属性值通过Integer.valueOf转换。value和param不能同时使用。


Servlet通过创建一个框架扩展服务器的能力，采用请求响应模式提供WEB服务。当客户机发送请求至服务器时，服务器将请求信息发送给Servlet，Servlet生成响应内容并将其传送给服务器，然后再由服务器将响应返回给客户端。


创建Servlet

所有的Servlet必须直接或间接地实现javax.servlet.Servlet接口，这样才能在Servlet服务器或称为Servlet容器，Servlet引擎上运行，更简单的方法是继承HttpServlet类。

过滤器

过滤器（Filter）是一个可以传送请求或修改响应的对象。过滤器并不是Servlet，他们并不实际创建一个请求。他们是请求到达一个Servlet前的预处理程序，在响应离开Servlet后的后台处理程序。一个过滤器可以做的工作：

  • 在一个Servlet被调用前截获该调用。
  • 在一个Servlet被调用前检查请求。
  • 修改实际请求中提供了可定制请求对象的请求头和请求数据。
  • 修改实际响应中提供了可定制响应对象的响应头和响应数据。
  • 在一个Servlet被调用之后截获该调用。

一个过滤器实现java.servlet.Filter接口并定义其3个方法：

  • init：在过滤器执行service前被调用，以设置过滤器的配置对象
  • destroy：在过滤器执行service后被调用
  • doFilter：执行实际的过滤工作

监听器

监听器（Listener）可以监听客户端的请求，服务端的操作等。通过监听器，可以自动激发一下操作，如监听在线用户数量，当增加一个HttpSession时，加1。

常用的监听接口：

  • ServletContextAttributeListener：监听ServletContext属性的操作，比如增加，删除，修改。
  • ServletContextListener：监听ServletContext。当创建ServletContext时，激发contextInitialized方法；当销毁时，激发contextDestroyed方法。
  • HttpSessionListener：监听HttpSession的操作。当创建一个Session时，激发sessionCreated方法；当销毁一个Session时，激发sessionDestroyed方法。
  • HttpSessionAttributeListener：监听HttpSession中的属性的操作。当Session中增加一个属性时，激发attributeAdded方法；当在Session中删除一个属性时，激发attributeRemoved方法；当在Session属性被重新设置时，激发attributeReplaced方法。

自定义JSP标签

一种封装其他动态类型的机制，扩展了JSP语言。自定义标签通常发布在标签库中，该标签库定义了一个自定义标签集并包含实现标签的对象。一些功能可以通过自定义标签来实现，包括对隐藏对象的操作，处理表单，访问数据库及其他企业级服务，如e-mail、目录服务、处理流控制。

    过滤器

        2018-03-14    

    过滤器

    使用下面的过滤器可以计算执行某个Servlet所用时间并记录在日志中。

    TimerFilter.java 

    package vip.chencheng.learnJavaWeb;

import java.io.*;

import javax.servlet.*;
import javax.servlet.http.*;

public class TimerFilter implements Filter {
	private FilterConfig config = null;

	public void destroy() {
		config = null;
		
	}

	public void doFilter(ServletRequest request, ServletResponse response,
			FilterChain chain) throws IOException, ServletException {
		long before=System.currentTimeMillis();
		chain.doFilter(request, response);
		long after = System.currentTimeMillis();
		String name="";
		if(request instanceof HttpServletRequest){
			name=((HttpServletRequest) request).getRequestURI();
		}
		config.getServletContext().log(name+":"+(after-before)+"ms");
		
	}

	public void init(FilterConfig config) throws ServletException {
		this.config=config;
		
	}

}

    实现过滤器，必须重写doFilter()方法。

    还要在web.xml文件中用<filter>标签部署他： 

      <filter>
  <filter-name>timerFilter</filter-name>
  <filter-class>vip.chencheng.learnJavaWeb.TimerFilter</filter-class>
  </filter>

    OlineCounter.java创建一个关于在线人数的类 

    package vip.chencheng.learnJavaWeb;

public class OnlineCounter {
	private static long online = 0;
	public static long getOnline(){
		return 0;
	}
	
	public static void raise(){
		online++;
	}
	
	public static void reduce(){
		online--;
	}

}
    OnOnlineCounterListener.java重写sessionCreated()方法，调用上面OnlineCounter的方法，在创建和销毁Session对象时改变在线人数。 
    package vip.chencheng.learnJavaWeb;

import javax.servlet.http.HttpSessionEvent;
import javax.servlet.http.HttpSessionListener;

public class OnlineCounterListener implements HttpSessionListener {

	public void sessionCreated(HttpSessionEvent hse) {
		// TODO Auto-generated method stub
		OnlineCounter.raise();
	}

	public void sessionDestroyed(HttpSessionEvent hse) {
		// TODO Auto-generated method stub
		OnlineCounter.reduce();
	}

}

    使用此过滤器要在web.xml文件用<listener>标签部署它 

      <listener>
  <listener-class>vip.chencheng.learnJavaWeb.OnlineCounterListener</listener-class>
  </listener>

    自定义JSP标签

    自定义JSP标签

    标签处理类必须实现Tag接口的doStartTag和doEndTag方法。当JSP引擎碰到tag标签的开头时，doStartTag调用，因为tag没有body，此方法返回SKIP_BODY。当JSP引擎碰到tag标签的结尾时，doEndTag调用，如果还有页面要计算，返回EVAL_PAGE，否则返回SKIP_PAGE。

    一般不必直接实现Tag接口，而是实现TagSupport类。

    SimpleTag.java 

    package vip.chencheng.learnJavaWeb;

import javax.servlet.jsp.JspException;
import javax.servlet.jsp.JspTagException;
import javax.servlet.jsp.tagext.TagSupport;

public class SimpleTag extends TagSupport {
	public int doStartTag() throws JspException{
		try{
			pageContext.getOut().print("hello reader!");
			pageContext.getOut().print("<br>");
			pageContext.getOut().print("I'm from customTag");
		} catch(Exception e){
			throw new JspTagException("simpleTag: "+e.getMessage());
		}
		return SKIP_BODY;
	}
	
	public int doEndTag(){
		return EVAL_PAGE;
	}

}

    标签库文件简称TLD文件，用XML描述。TLD包括了标签库中所有标签的描述，一般被JSP服务器用来校验标签语法正确性，或者被JSP开发者用来开发新的标签。扩展名必须为.tld。放在web应用的WEB-INF目录或其子目录中。

    helloreader.tld 

    <?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE taglib
PUBLIC "-//Sun Microsystems, Inc.//DTD JSP Tag Library 1.1//EN"
"http://java.sun.com/j2ee/dtds/web-jsptaglibrary_1_1.dtd">
<taglib>
	<tlibversion>1.0</tlibversion>
	<jspversion>1.1</jspversion>
	<shortname>hellodemo</shortname>
	<tag>
		<name>hello</name>
		<tagclass>vip.chencheng.learnJavaWeb.SimpleTag</tagclass>
		<bodycontent>empty</bodycontent>
	</tag>
</taglib>

    配置web.xml，使用自定义标签<taglib> 

     <jsp-config>  
       <taglib>  
  		<taglib-uri>/WEB-INF/ helloreader.tld</taglib-uri>
  		<!-- 加载标签，供页面使用 -->
  		<taglib-location>/WEB-INF</taglib-location>
	</taglib>  
</jsp-config>

    显示自定义标签。访问链接http://localhost/WebRoot/tagTest.jsp

    tagTest.jsp 

    <%@taglib uri="/WEB-INF/helloreader.tld" prefix="demo"  %>
<html>
  <head>
    <title>tag test</title>
  </head>
  
  <body> 
  <demo:hello />
  
  </body>
</html>

    标签库及描述文件

        2018-03-14    

    标签库涉及4个基本概念：

    • 标签（Tag）：标签是一种XML元素，通过标签可以使JSP网页变得简洁并且易于维护，还可以方便地实现同一个JSP文件支持多种语言版本。由于标签是XML元素，所以它的名称和属性都是大小写敏感的。
    • 标签库（Tag Library）：由一系列功能相似、逻辑上互相联系的标签构成的集合称为标签库。
    • 标签库描述文件（Tag Library Descriptor File）：标签库描述文件是一个XML文件，简称TLD文件，这个文件提供了标签库中类的JSP中对标签引用的映射关系。它是一个配置文件，和web.xml是类似的。
    • 标签处理类（Tag Handle Class）：标签处理类是一个Java类，这个类继承了TagSupport或者扩展了SimpleTag接口，通过这个类可以实现自定义JSP标签的具体功能。

    开发一个自定义标签，主要作用就是创建TLD文件和编写标签处理类。


    TLD的文件扩展名必须为.tld，而且必须放在当前WEB应用的WEB-INF目录或其子目录中。一个TLD的内容的开头必须遵守标准的XML开头，用于描述DTD和xml的版本，例如：
    <?xml version=”1.0″ encoding=”ISO-8859-1″ ?><!DOCTYPE taglib PUBLIC “-//Sun Microsystems, Inc.//DTD JSP Tag Library 1.2//EN” “http://java.sun.com/dtd/web-jsptaglibrary_1_2.dtd”>

    TLD必须以<taglib>来作为它的根元素，

    <taglib>的子元素如下表：

    1）<taglib>的子元素
    • Element：Description
    • tlib-version：Tag库的版本
    • jsp-version：Tag库所需要的jsp的版本
    • short-name：助记符，tag的一个别名（可选）
    • uri：用于确定一个唯一的tag库
    • display-name：被可视化工具（诸如Jbuilder）用来显示的名称（可选）
    • small-icon：被可视化工具（诸如Jbuilder）用来显示的小图标（可选）
    • large-icon：被可视化工具（诸如Jbuilder）用来显示的大图标（可选）
    • description：对tag库的描述（可选）
    • listener：参见下面listener元素
    • tag：参见下面tag 元素
    • Listener元素：一个tag库可能定义一些类做为它的事件侦听类，这些类在TLD中被称为listener 元素，jsp服务器将会实例化这些侦听类，并且注册它们。Listener元素中有一个叫listener-class的子元素，这个元素的值必须是该侦听类的完整类名。
    2）Tag元素
    每个tag元素在tag库中都要指出它的名字、类名、脚本变量、tag的属性。其中脚本变量的值可以直接在TLD中定义或通过tag附加信息的类来取得。每个属性描述了这个属性是否可以省略，它的值是否可以通过<%= …%>这样的JSP语法来获得，以及属性的类型。每一个tag在TLD中对应一个tag元素，下表是tag元素的子元素：
    • name：独一无二的元素名
    • tag-class：Tag标签对应的tag处理类
    • tei-class：javax.servlet.jsp.tagext.TagExtraInfo的子类，用于表达脚本变量（可选）
    • body-content：Tag标签body的类型
    • display-name：被可视化工具（诸如Jbuilder）用来显示的名称（可选）
    • small-icon：被可视化工具（诸如Jbuilder）用来显示的小图标（可选）
    • large-icon：被可视化工具（诸如Jbuilder）用来显示的大图标（可选）
    • description：此tag标签的描述
    • variable：提供脚本变量的信息（同tei-class）(可选)
    • attribute：Tag标签的属性名
