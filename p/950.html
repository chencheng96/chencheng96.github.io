mongodump/mongorestore备份和恢复数据库

备份数据

打开命令提示符，进入安装好的MongoDB的bin目录下，输入命令： 

D:\Program Files\MongoDB\Server\3.2\bin>mongodump -h 127.0.0.1:27017 -d mytest -o f:\mongodb\dump
其中：
  • -h：MongDB所在服务器地址，例如：127.0.0.1，当然也可以指定端口号
  • -d：需要备份的数据库实例，例如：mytest （数据库名称，可以备份该数据库下所有集合）
  • -o：备份的数据存放位置，例如：f:\mongodb\dump，在备份完成后，系统自动在dump目录下建立一个mytest目录，这个目录里面存放该数据库实例的备份数据。

数据恢复

使用mongorestore命令，进行数据恢复，同样打开命令提示符，进入安装好的MongoDB的bin目录下，然后输入命令 

D:\Program Files\MongoDB\Server\3.2\bin>mongorestore -h 127.0.0.1:27017 -d mytest f:\mongodb\dump\mytest
  • -h：MongoDB所在服务器地址
  • -d：需要恢复的数据库实例，例如：mytest。当然这个名称也可以和备份时候的不一样，比如test2，这样就会创建一个新的集合。
  • --directoryperdb：备份数据所在位置，例如：f:\mongodb\dump\mytest
  • --drop： 恢复的时候，先删除当前数据，然后恢复备份的数据。就是说，恢复后，备份后添加修改的数据都会被删除，慎用！

mongoexport/mongoimport数据导入导出

导出工具：mongoexport

mongoexport工具可以把一个collection导出成JSON格式或CSV格式的文件。可以通过参数指定导出的数据项，也可以根据指定的条件导出数据。
mongoexport -d dbname -c collectionname -o file --type json/csv -f field
参数说明：
  • -d ：数据库名
  • -c ：collection名
  • -o ：输出的文件名，可以指明路径
  • --type ： 输出的格式，默认为json
  • -f ：输出的字段，如果-type为csv，则需要加上-f "字段名"

示例： 

D:\Program Files\MongoDB\Server\3.2\bin>mongoexport -d mytest -c first -o f:\mongodb\dump\mytest_first.json --type json

数据导入：mongoimport

mongoimport -d dbname -c collectionname --file filename --headerline --type json/csv -f field
参数说明：
  • -d ：数据库名
  • -c ：collection名
  • --type ：导入的格式默认json
  • -f ：导入的字段名
  • --headerline ：如果导入的格式是csv，则可以使用第一行的标题作为导入的字段
  • --file ：要导入的文件

示例： 

D:\Program Files\MongoDB\Server\3.2\bin>mongoimport -d mytest -c first --file f:\mongodb\dump\mytest_first.json --type json
